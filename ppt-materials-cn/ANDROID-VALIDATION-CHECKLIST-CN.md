# Android 验收检查清单（中文）

本清单用于当前原生 Android 主线 `app/` 的短验。

目标：

- 验证“通知 + 生词本 + 学习反馈”这条用户主链是否可用
- 即使模拟器中仍有 warning，也先确认当前 beta 版本在实际功能上是否可接受

## 范围

本清单是刻意收窄的，只覆盖最重要的用户主链：

1. 开启推送
2. 收到单词通知
3. 标记认识或不认识
4. 验证生词本行为
5. 验证解锁刷新行为
6. 验证关闭推送后的行为

## 前置条件

- 安装最新 debug 或 beta 包
- 首次启动时授予通知权限
- 至少打开过一次 app，并确认首页正常显示
- 至少保持一个词库已启用

## 通过标准总览

如果满足以下全部条件，则当前版本可视为功能上基本可接受：

- 开启推送后能出现单词通知
- 点击 `不认识` 后，该词会进入生词本
- 点击 `认识` 后，不会触发通知刷屏
- 解锁时最多只触发一次合理刷新
- 关闭推送后，不再生成新的通知

## 测试 1：首页状态

步骤：

1. 打开 app
2. 停留在首页
3. 观察推送状态、当前词库、生词本数量

期望：

- 首页加载不崩溃
- 推送开关可见且可操作
- 当前词库文案正常
- 生词本数量可见

记录：

- `Pass` 或 `Fail`
- 记录当前显示的生词本数量

## 测试 2：开启推送

步骤：

1. 打开推送开关
2. 等待几秒

期望：

- 出现成功提示
- 出现单词通知
- 通知中的当前词可读
- 通知包含 `认识` 和 `不认识`

记录：

- 是否出现通知
- 从开启到出现通知大约用了多久

## 测试 3：不认识 -> 生词本

步骤：

1. 在通知里点击 `不认识`
2. 回到 app
3. 切换到生词本页

期望：

- 当前通知被正确处理
- 生词本页正常打开
- 刚刚点击的单词能在生词本中看到
- 如果该词原本不在生词本里，数量应增加

记录：

- 你点击的是哪个词
- 它是否进入生词本
- 数量是否变化

## 测试 4：生词本显示

步骤：

1. 保持在生词本页
2. 检查当前单词卡片和侧边单词列表
3. 如果有多个词，点一到两个词切换

期望：

- 生词本标题正常显示
- 单词卡片能显示：单词、音标、释义、例句
- 点击侧边词条后，详情区域会更新
- 发音按钮不会导致 app 崩溃

记录：

- 页面显示是否正常
- 是否存在缺失字段，如词库、词性、音标、释义等

## 测试 5：认识 -> 解锁刷新

步骤：

1. 回到一个仍然有效的通知
2. 点击 `认识`
3. 锁屏
4. 再解锁一次

期望：

- 当前通知会被正确替换或消失
- 解锁后只出现一次下一词刷新
- 不会在短时间内多次重复刷通知

记录：

- 是否只刷新了一次
- 是否异常重复同一个词

## 测试 6：亮屏与解锁时机

步骤：

1. 保持推送开启
2. 熄屏
3. 只点亮屏幕但不完全解锁
4. 观察是否立刻出现新词
5. 再完成解锁

期望：

- 单纯亮屏不应造成明显的重复刷新噪音
- 完整解锁后可触发一次符合当前逻辑的刷新

记录：

- 只亮屏时发生了什么
- 完整解锁时发生了什么

## 测试 7：关闭推送

步骤：

1. 回到首页
2. 关闭推送开关
3. 等待并再执行一次锁屏 / 解锁

期望：

- 出现成功提示
- 不再出现新的通知
- 关闭后普通解锁不应再生成新通知

记录：

- 通知是否完全停止

## 测试 8：重启恢复

步骤：

1. 开启推送
2. 重启设备
3. 解锁并等待片刻
4. 如可行，再关闭推送并再次重启

期望：

- 推送开启状态下，重启后调度能够恢复
- 推送关闭状态下，重启后不应被偷偷重新激活

记录：

- 开启推送后重启的结果
- 关闭推送后重启的结果

## 当前已知风险

在写这份清单时，模拟器中仍可能看到前台服务类型 warning。

解读原则：

- 如果真机上的用户可见行为正常，应把它视为兼容性尾问题，而不是立刻阻断发布的致命问题
- 如果真机上也出现同样 warning，且影响通知稳定性，则 `WordService` 启动路径仍是后续工程重点

## 最终判断模板

测试结束后可以按下面的形式总结：

- 首页状态：`Pass` / `Fail`
- 开启推送：`Pass` / `Fail`
- 不认识 -> 生词本：`Pass` / `Fail`
- 生词本显示：`Pass` / `Fail`
- 认识 -> 解锁刷新：`Pass` / `Fail`
- 亮屏 / 解锁时机：`Pass` / `Fail`
- 关闭推送：`Pass` / `Fail`
- 重启恢复：`Pass` / `Fail`

整体结论：

- `Accept for beta`
- `Accept with known warning`
- `Do not accept yet`
