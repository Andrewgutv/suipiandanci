# 🔧 原生插件测试指南

## ⚠️ 重要提示

**原生插件功能必须在以下环境中测试：**
- ✅ 云打包的 APK
- ✅ 自定义调试基座
- ❌ **不支持**：浏览器、H5、小程序、标准基座

---

## 📋 问题诊断

### 你看到的错误："显示失败,请确保已安装原生插件"

**原因**：原生插件未正确加载。通常是因为：
1. 使用了标准基座运行（不支持原生插件）
2. 在浏览器中测试（不支持原生插件）
3. 插件配置问题（已修复）

---

## ✅ 解决方案

### 方法1：云打包 APK（最简单，推荐）

**步骤：**

1. **打开 HBuilderX**
   - 确保版本 >= 3.6.0

2. **检查配置**
   - 打开 `manifest.json`
   - 确认 "nativePlugins" 包含 "WordLockScreen"

3. **云打包**
   ```
   发行 → 原生App-云打包 → 选择Android → 勾选"使用原生插件" → 打包
   ```

4. **下载并安装**
   - 等待 3-5 分钟
   - 下载 APK
   - 安装到手机

5. **测试**
   - 打开应用
   - 点击"🚀 开启锁屏单词"
   - 应该会显示"插件加载成功"
   - 确认后会显示锁屏通知

---

### 方法2：制作自定义调试基座（用于开发调试）

如果你想在开发过程中频繁测试，需要制作自定义基座：

**步骤：**

1. **制作自定义基座**
   ```
   发行 → 原生App-制作应用wgt包 → 选择"自定义调试基座" → 打包
   ```

2. **下载基座**
   - 等待打包完成
   - 下载自定义基座

3. **安装基座到手机**
   - 通过 USB 连接手机
   - 安装自定义基座 APK

4. **运行到自定义基座**
   ```
   运行 → 运行到手机或模拟器 → 选择设备 → 运行到Android基座 → 选择自定义基座
   ```

5. **测试**
   - 现在可以正常使用原生插件了

---

### 方法3：本地打包（需要 Android Studio）

1. **本地打包**
   ```
   发行 → 原生App-本地打包 → 生成本地打包资源
   ```

2. **在 Android Studio 中打开**
   - 导入生成的项目
   - 复制 nativeplugins 目录
   - Build → Generate Signed APK

## 🧪 测试流程

### 1. 确认环境

```javascript
// 应用会自动检测并显示提示
// 首次打开应该看到"插件加载成功"的提示
```

### 2. 测试锁屏功能

1. **打开应用**
   - 点击"🚀 开启锁屏单词"

2. **确认对话框**
   - 点击"开启"

3. **查看效果**
   - 应该显示"已显示在锁屏通知中"
   - 锁屏手机
   - 在锁屏界面查看单词卡片

### 3. 测试按钮交互

在锁屏通知中：
- 点击"认识" → 应该显示 Toast 提示
- 点击"不认识" → 应该显示 Toast 提示
- 点击通知 → 打开应用

## 🛠️ 已修复的问题

### 1. 插件名称不一致
- ✅ package.json: `"id": "WordLockScreen"`
- ✅ manifest.json: `"WordLockScreen"`
- ✅ 前端调用: 自动尝试多种名称

### 2. 添加插件检测
- ✅ 应用启动时自动检测插件
- ✅ 显示加载状态提示
- ✅ 详细的错误信息

### 3. 多名称兼容
- ✅ 自动尝试多种可能的插件名称
- ✅ 提高兼容性

## 💡 建议

### 快速测试（推荐）
